# Job Statement Templates

## Canonical Format

```
When [specific circumstance],
I want [desired progress],
So I can [expected outcome].
```

**Rules:**
- [circumstance] describes WHEN the job happens, not who the person is
- [progress] describes what the person wants to ACCOMPLISH, never a feature or solution
- [outcome] describes the RESULT that makes the progress valuable

## Examples — Generic B2B Context

### Operations Manager — Onboarding a New Vendor

**Functional:**
> When I need to onboard a new service provider for a client engagement,
> I want to confirm their qualifications meet the project requirements,
> So I can start the engagement without compliance risk.

**Emotional:**
> When I am reviewing a provider's profile before a high-stakes engagement,
> I want to feel confident that the evaluation is thorough and traceable,
> So I can approve them without second-guessing myself.

**Social:**
> When presenting a vendor decision to my client,
> I want to show a clear, auditable trail of the evaluation,
> So I can be seen as a rigorous partner, not a rubber stamp.

### Service Provider — Availability for New Work

**Functional:**
> When I complete a required compliance update,
> I want to know immediately if I am cleared for new assignments,
> So I can accept work without waiting days for a response.

**Emotional:**
> When I have been blocked from work due to a pending review,
> I want to understand exactly what is missing,
> So I don't feel stuck in a black box.

### Platform Operator — Reviewing an Application

**Functional:**
> When an automated check returns an inconclusive result,
> I want to quickly see which specific data point triggered the gap,
> So I can decide whether to request a recheck or escalate manually.

## Validation Checklist

Before finalizing a job statement, verify:
- [ ] The circumstance describes a real moment, not an abstract persona
- [ ] The progress is free of solution references ("use the app", "click the button")
- [ ] The outcome connects the progress to real value for the customer
- [ ] The job could be accomplished with different solutions (solution-independence test)
- [ ] It is not a double job (two jobs merged into one statement)
