# Output Format: HTML Story

## What it is

A navigable HTML page generated by `html_story_builder.py` with a complete narrative: executive TL;DR, SWD-style charts, context/finding/so-what per section, and an Appendix with queries. Standard output for data analyses and internal reports.

## When to use

- Data-based story with 1–5 charts
- Analysis that needs to be shared asynchronously
- Output from data research + interpretation
- When the audience will read it independently (not a live presentation)

## Generation

```python
import sys
sys.path.insert(0, 'analysis/scripts')
from html_story_builder import build_html_story

CONTRACT = {
    "chart_type": "...",
    "title": "main message (the finding)",
    "subtitle": "context",
    "source": "origin",
    "collected_at": "YYYY-MM-DD",
    "highlight": "main_label",
    "output_path": "analysis/<slug>",
    "data": { },
    "narrative": {
        "context": "why this data matters",
        "so_what": "implication for the business/product",
        "next_steps": ["...", "..."],
        "appendix_query": "complete query"
    }
}

path = build_html_story(CONTRACT)
print(f"Story: {path}")
```

Run: `python3 /tmp/story_agent_<slug>.py`

## Structure of the generated HTML

```html
<section class="tldr-block">
  <!-- TL;DR — inserted by the tldr skill after charts are generated -->
  <!-- Max 10 lines — entry point of the reading -->
</section>

<!-- Charts (max 5) with inline narrative -->
<section class="chart-section">
  <h2>[Title = chart message]</h2>
  <p class="context">[context]</p>
  <img src="..." />
  <p class="so-what">[implication]</p>
</section>

<!-- Appendix -->
<section class="appendix">
  <h2>Appendix — Data and Sources</h2>
  <!-- Queries/sources — mandatory -->
</section>
```

## Slug and location

- **Path:** `analysis/<slug>.html`
- **Slug:** `YYYY-MM-DD-<topic>` — e.g., `2026-03-06-churn-analysis`

## Post-generation — mandatory

**1. TL;DR** — invoke the `tldr` skill with the full narrative and insert as the first section:
```
Skill: "tldr"
Context: full narrative (context + findings + so what from each chart)
Audience: profile identified in Phase 0
```

**2. UX polish** — invoke `frontend-design` as reviewer before delivering:
- Typographic hierarchy, spacing, visual weight, layout consistency
- Do NOT change: accent color, neutral palette, no pie charts, title = message

## Rules

- **Appendix mandatory** — query/source for every number
- **Max 5 charts** — if more are needed, question whether it's a story or a dashboard
- **Titles = messages** — never a description of the data
- **`[DATA NEEDED: ...]`** for gaps — never invent
