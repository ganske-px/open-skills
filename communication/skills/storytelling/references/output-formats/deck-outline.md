# Output Format: Deck Outline

## What it is

A structured slide script — not the slide itself, but the content of each slide with a title, main message, and visual support. Allows reviewing the narrative before entering the design phase.

## When to use

- Audience will receive a live presentation (live deck)
- Story has 3+ distinct messages
- Stakeholder approval is needed before creating the final visual
- Any story going into PowerPoint, Google Slides, or Keynote

## Structure template

```markdown
## DECK: [presentation title = value proposition in 1 sentence]

**Audience:** [exec / squad / customer / all-hands]
**Time:** [X minutes]
**Objective:** [approval / alignment / information / inspiration]

---

### Slide 1 — OPENING
**Title:** [emotional hook or question]
**Content:** customer story OR impact data OR provocative question
**Visual:** photo / quote / large number
**Time:** 1–2 min

---

### Slide 2 — CONTEXT / SITUATION
**Title:** [stable context sentence]
**Content:** [max 3 bullets — what the audience needs to know to understand the problem]
**Visual:** diagram / timeline / simple map
**Time:** 1–2 min

---

### Slide 3 — PROBLEM / COMPLICATION
**Title:** [the problem in 1 sentence — data in title when possible]
**Content:** main data point + context
**Visual:** chart (follow data-insight.md)
**Time:** 2–3 min

---

### Slide N — [ARGUMENT / FINDING / BET]
**Title:** [slide message — never a description]
**Content:** [data or argument supporting the title]
**Visual:** chart / diagram / table
**Time:** 2 min

---

### FINAL Slide — NEXT STEPS
**Title:** [specific call to action]
**Content:**
- Action 1 — owner — deadline
- Action 2 — owner — deadline
- Action 3 — owner — deadline
**Visual:** simple table or bullets
**Time:** 1–2 min

---

### APPENDIX Slide — DATA AND SOURCES
**Content:** queries, sources, methodology
**Rule:** mandatory for any presentation with data
```

## Quality rules

- **Each slide title = the message** (not a description of the content)
- **1 message per slide** — if you need two titles, use two slides
- **Visual serves the title** — not the other way around
- **Max 3 bullets per slide** — if you need more, the slide is trying to do too much
- **Deck must work on its own** — an audience member who wasn't in the room should understand the narrative from the titles alone

## Adaptation by audience (see `audience-adaptation.md`)

| Audience | Recommended slides | Time |
|---|---|---|
| Exec / C-level | 5–7 slides | 10–15 min |
| Squad / team | 8–15 slides | 20–30 min with debate |
| All-hands | 10–20 slides | 20–30 min with Q&A |
| Customer | 5–10 slides | 15–20 min |
