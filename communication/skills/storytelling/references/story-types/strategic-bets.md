# Story Type: Strategic Decisions / Bets

## When to use
- Present a product bet for approval
- Justify why we're prioritizing X over Y
- Recommend a direction among real alternatives
- "We're betting on [path] because..."
- Decision with quarterly or greater impact

## Framework: Pyramid Principle (Minto) + Decision Memo

Strategic decisions need an explicit recommendation at the top. Executive audiences have no patience for conclusions at the end. The structure is: say what you think → prove it → anticipate objections.

### Required structure

```
1. RECOMMENDATION (always first)
   → "We recommend [concrete action] because [main reason in 1 sentence]"
   → Not "we'll present options and decide together" — that's not a recommendation
   → Who decides: name explicitly (RAPID: who Recommends, Approves, has Input, etc.)

2. CONTEXT (minimum necessary)
   → What problem motivated the decision
   → Relevant data: opportunity size, urgency, market signal
   → Max 3 points — executive audience already knows the context

3. SUPPORTING ARGUMENTS (3, maximum)
   → Each argument supported by data or evidence
   → Order: strongest to weakest
   → Each argument answers "why this option, not the others"

4. ALTERNATIVES CONSIDERED AND REJECTED
   → Minimum 2 real alternatives (not strawmen)
   → Why each was rejected (criterion, not opinion)
   → Shows analytical rigor — audience trusts more when they see you thought about other options

5. RISKS AND HOW WE MANAGE THEM
   → Max 3 main risks
   → For each risk: estimated probability + mitigation
   → [DATA NEEDED: ...] for real gaps

6. NEXT STEPS AND SUCCESS CRITERION
   → What happens if approved: 3 actions with owner and deadline
   → How we'll know it was the right decision: specific metric with deadline
   → Reversal criterion: when we revisit the bet
```

## Skills to invoke

| When | Skill |
|---|---|
| There are multiple options to compare | `decision-making` — Decision Matrix with weighted criteria |
| Involves product prioritization | `product-decision` — OST, bet analysis |
| Supporting data for the recommendation | `data-research` |
| Customer voice to back the bet | `synthesize-research` + `research-feedback` |
| Success probability analysis | `probability-analysis` |

## Non-negotiable rules

- **Recommendation at the top.** Always. If you don't have a recommendation, you're not ready to present
- **Real alternatives.** If the alternatives look too weak, the audience will suspect the process
- **Data or `[DATA NEEDED]`.** Never estimate without declaring confidence level
- **Reversal criterion.** Every bet has an exit condition — naming it demonstrates maturity
- **No "obviously".** Nothing is obvious to whoever is approving

## Collection questions

1. "What is the recommendation in one sentence? (if you can't formulate it, you're not ready)"
2. "What are the real alternatives that were considered?"
3. "What is the strongest data supporting the recommendation?"
4. "What is the biggest risk? What is the mitigation?"
5. "Who needs to approve? What is the decision deadline?"
6. "If it goes wrong, how will we know and what will we do?"

## Recommended output

| Audience | Preferred format |
|---|---|
| Board / C-level | `exec-memo` (1–2 pages) — async reading before the meeting |
| Decision meeting | `deck-outline` (5–7 slides) as visual support for the memo |
| Squad / internal alignment | `deck-outline` with focus on arguments + debate on alternatives |

## Decision Matrix template (when using `decision-making`)

| Criterion | Weight | Option A | Option B | Option C |
|---|---|---|---|---|
| User impact | 40% | 8 | 6 | 4 |
| Execution speed | 30% | 5 | 8 | 7 |
| Technical risk | 30% | 3 | 6 | 8 |
| **Weighted score** | | **5.6** | **6.8** | **6.1** |

## Anti-patterns

- Presenting data without arriving at a recommendation ("we'll let the group decide")
- Rejected alternatives without criterion — looks like manipulation
- Risks without mitigation — audience will get stuck on the list of problems
- Slides with too much text — memo is for reading, slide is for presenting — don't mix them
