# Framework: Value Proposition

## What it is

The central message the presentation must convey. It must be built BEFORE any slide, visual, or data. If you cannot formulate the value prop in 1 sentence, you are not ready to present.

## Base formula

```
For [specific audience],
who faces [concrete problem/job],
[product/initiative/decision] offers [main benefit].
Unlike [current alternative],
[specific differentiator].
```

## Adaptation by audience

| Audience | Primary emphasis | Tone |
|---|---|---|
| Exec / C-level | ROI, impact in numbers, decision speed | Direct, no excessive context |
| Engineering | Trade-offs, feasibility, real technical constraints | Honest about uncertainties |
| Sales / Commercial | Customer benefits, social proof, use cases | Customer-outcome oriented |
| Internal team / Squad | Mission clarity, impact on daily work | Collaborative, invites debate |
| All-hands | Purpose, connection to the larger vision | Inspiring, emotional allowed |

## Checklist before proceeding

- [ ] Does the message fit in 1 sentence?
- [ ] Is it adapted for THIS specific audience?
- [ ] Does it include a concrete benefit (not "improve X" but "reduce X from 4h to 30min")?
- [ ] Is there supporting data or real customer voice? If not: `[DATA NEEDED: ...]`

## How to use it in the story

The value prop becomes:
- **Presentation title** — not a description of the content
- **Opening line** — before any context
- **Closing slide** — the message the audience should remember

## Anti-patterns

- "Our product improves user experience" — no number, no audience, no problem
- Value prop that contradicts what the data shows — audiences notice the inconsistency
- Adapting the message ON THE FLY during the presentation — signals lack of preparation
