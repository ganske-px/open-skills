# Framework: Pyramid Principle (Minto)

## When to use

Every time the audience needs to reach a conclusion or make a decision. Especially for: exec, C-level, any situation where time is scarce and the recommendation needs to be clear.

## Core principle

**Conclusion first. Always.**

The "bottom-up" structure (building the argument until reaching the conclusion) is natural for whoever analyzed the problem — but torturous for whoever needs to decide. Invert it: give the conclusion, then prove it.

## Structure

```
TOP — Answer / Recommendation
  "I recommend [action] because [reason in 1 sentence]"

SECOND LEVEL — Supporting arguments (max 3)
  Argument 1 — strongest
  Argument 2
  Argument 3 — weakest (but still necessary)

THIRD LEVEL — Evidence and data
  For each argument: data, case, analysis that sustains it
```

## SCQA — How to introduce context (before the pyramid)

For situations where the audience needs context before the recommendation:

```
S — Situation (stable, shared context)
  "The approval process today has 4 manual steps"

C — Complication (what changed or what the problem is)
  "With growth, we're approving 3x more contracts with the same team"

Q — Question (the question that naturally arises)
  "How do we scale without growing the team proportionally?"

A — Answer (your recommendation — top of the pyramid)
  "I recommend automating steps 2 and 4 using [solution]"
```

## Grouping rules

The supporting arguments must be:
- **MECE** (Mutually Exclusive, Collectively Exhaustive): no overlap, no gap
- Parallel in form: all nouns, all action verbs, etc.
- Ideally an odd number (3 is ideal)

## Examples

### Weak (bottom-up, no pyramid)
"We analyzed the approval data and saw that... we also identified that... furthermore... therefore, we recommend..."

### Strong (pyramid)
"We recommend automating steps 2 and 4 of the process. This reduces approval time by 60%, eliminates the main growth bottleneck, and has been validated with the technical team as feasible in 6 weeks."

## How to use for slides

Each slide = 1 level of the pyramid:
- Slide 1: Recommendation (slide title = the recommendation)
- Slides 2–4: One argument per slide (title = the argument, not the description)
- Last slide: Next steps + success criterion

## Anti-patterns

- "We'll present the data and decide together" — not a pyramid, it's lack of recommendation
- Supporting argument that doesn't directly sustain the top recommendation
- More than 3 main arguments — audience doesn't retain
- SCQA too long — if the situation is known, jump straight to Complication
